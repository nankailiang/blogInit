## 内存泄漏和内存溢出的区别

### 内存泄漏（Memory Leak）
- **定义**：对象无实际用途但仍被引用，垃圾回收器无法回收，导致内存逐渐耗尽
- **特点**：内存使用量逐渐增加，最终可能导致内存溢出

### 内存溢出（OOM）
- **定义**：内存不足（如堆内存满、栈溢出），无法为新对象分配内存，抛出内存溢出异常
- **特点**：突然发生，程序崩溃

### 关系
内存泄漏是内存溢出的重要原因之一。

## 常见的内存泄漏场景
1. **静态集合持有对象**：如static List<User> list，添加对象后不清理
2. **匿名内部类/非静态内部类持有外部类引用**：如Handler持有Activity引用，导致Activity无法回收
3. **资源未关闭**：如IO流、数据库连接、Socket未关闭，占用内存资源
4. **ThreadLocal未清理**：ThreadLocal的Entry持有线程引用，线程长期存活导致对象无法回收
