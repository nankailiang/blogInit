## 线程池的核心参数
ThreadPoolExecutor的7个核心参数：

### 1. 核心线程数（corePoolSize）
- **定义**：线程池长期保持的线程数（空闲时不销毁）
- **作用**：处理常规任务的基础线程数

### 2. 最大线程数（maximumPoolSize）
- **定义**：线程池允许的最大线程数（核心线程满+工作队列满时，创建临时线程，最多到该值）
- **作用**：处理突发任务的最大线程数

### 3. 空闲线程存活时间（keepAliveTime）
- **定义**：临时线程（超过核心线程数的线程）空闲后的存活时间，超时销毁
- **作用**：控制临时线程的生命周期

### 4. 时间单位（unit）
- **定义**：keepAliveTime的时间单位（如TimeUnit.SECONDS）
- **作用**：指定存活时间的单位

### 5. 工作队列（workQueue）
- **定义**：核心线程满时，任务等待的队列（如LinkedBlockingQueue、ArrayBlockingQueue）
- **作用**：缓冲待执行的任务

### 6. 线程工厂（threadFactory）
- **定义**：创建线程的工厂（自定义线程名称、优先级等）
- **作用**：定制线程的创建过程

### 7. 拒绝策略（handler）
- **定义**：任务队列满+最大线程数满时，处理新任务的策略（4种默认策略）
- **作用**：处理任务过载的情况
